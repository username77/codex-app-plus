# Test Plan: 后端网络/RPC 通信优化

## 测试策略

本优化主要涉及 Rust 后端的并发模式和 I/O 行为，测试以 Rust 单元测试和集成测试为主。

## 测试项

### T1: spawn_blocking 包装验证

**目标**: 确保 async 命令正确调用 spawn_blocking 并传递结果

**测试方式**: 编译通过即为基本验证（类型系统保证），辅以现有 Tauri 命令的功能测试

**验收标准**:
- `cargo build` 编译无错误
- 现有 `process_manager/tests` 全部通过
- 前端调用 auth/provider 命令的行为不变

### T2: 终端输出节流

**目标**: 验证节流逻辑正确性

**测试用例**:
1. **低频输出不延迟**: 两次输出间隔 > 16ms 时，每次都立即 emit
2. **高频输出合并**: 短时间内多次输出被合并为一次 emit
3. **缓冲区满立即 flush**: 累积数据达到 4KB 时不等待时间窗口
4. **退出时 flush 残留**: 循环退出后残留数据被正确 emit
5. **空数据不 emit**: decode 返回 None 时不累积空字符串

**测试方式**: 由于 `spawn_output_thread` 涉及线程和 IPC，建议将节流逻辑提取为可测试的纯函数或结构体。

### T3: 文件锁机制

**目标**: 验证跨进程文件锁正确性

**测试用例**:
1. **正常写入成功**: 获取锁 → 写入 → 释放锁，文件内容正确
2. **锁释放后可再次获取**: 连续两次 write_live_files 调用成功
3. **原子写入回滚保持**: 写入第二个文件失败时回滚第一个文件，文件锁正确释放

**测试方式**: Rust 单元测试，使用临时目录

### T4: 回归测试

**目标**: 确保优化不破坏现有功能

**测试方式**:
- `cargo test --manifest-path src-tauri/Cargo.toml` 全部通过
- `pnpm test` 前端测试全部通过
- 手动验证：启动 app → 连接 app-server → 发送消息 → 终端操作 → 切换 provider
